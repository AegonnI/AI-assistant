Multi-LLM Coefficient Extractor
Этот скрипт автоматически извлекает структурированные данные (коэффициенты) из текстовых файлов, JSON и PDF, используя различные провайдеры LLM (Groq, Google Gemini, OpenRouter).
🚀 Особенности

* Поддерживаемые форматы: Чтение данных из .txt, .json и .pdf.
* Гибкий выбор моделей: Работа с API Groq, Google (Gemini) и OpenRouter.
* Умный парсинг: Автоматическое разбиение длинных текстов на части и извлечение JSON-объектов из ответов моделей.
* Сбор данных: Скрипт проходит по частям текста и сохраняет первое найденное непустое значение для каждого искомого коэффициента.

🛠 Требования
Для работы с PDF установите библиотеку pypdf:

pip install pypdf requests google-generativeai groq

📋 Использование1. Подготовка функции
Основная точка входа — функция extract_coeffs. Она принимает путь к файлу и список ключей, которые нужно найти.

🔍 Как это работает

   1. Загрузка: Скрипт считывает текст из файла.
   2. Чанкинг: Текст делится на части (по умолчанию 6000 символов), чтобы соответствовать контекстному окну модели.
   3. Промптинг: Для каждого чанка формируется строгий запрос на возврат JSON.
   4. Слияние: Результаты всех чанков объединяются в один финальный словарь.

⚠️ Важно

* Убедитесь, что у вас есть соответствующие API-ключи для выбранного провайдера.

------------------------------

🔌 Поддерживаемые провайдеры (API)
Скрипт унифицирует вызов разных LLM через функцию api_request. Вы можете использовать следующие семейства моделей:
1. Groq (model_family="Groq")
Высокоскоростной инференс.

* Модель по умолчанию: qwen/qwen3-32b
* Особенности: Использует потоковую передачу (stream) для сборки ответа.
* Требуется: pip install groq

2. Google Gemini (model_family="google")

* Модель по умолчанию: gemini-2.5-flash
* Особенности: Стандартный вызов через SDK Google.
* Требуется: pip install google-generativeai

3. OpenRouter (model_family="openrouter")
Агрегатор, дающий доступ к сотням моделей (Llama, Mistral, Claude и др.) через единый интерфейс.

* Модель по умолчанию: mistralai/mixtral-8x7b-instruct
* Особенности: Работает через стандартные HTTP-запросы (requests).
* Параметры: Лимит в 300 токенов на ответ (настраивается в коде).
